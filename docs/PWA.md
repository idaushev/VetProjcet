# VetClinic PWA — Офлайн-поддержка

## Архитектура офлайн-режима

```
┌─────────────────────────────────────────────────────────────────┐
│  Первое посещение (онлайн)                                      │
│                                                                 │
│  Browser → Go Server → HTML/CSS/JS → Service Worker кэширует   │
│                                   → IndexedDB инициализируется  │
│                                   → Данные синхронизируются     │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  Последующие визиты (онлайн или офлайн)                        │
│                                                                 │
│  Browser → Service Worker (из кэша) → HTML/CSS/JS              │
│         → app.js interceptor        → IndexedDB (данные)       │
│         → фоновая синхронизация     → Go Server (если онлайн)  │
└─────────────────────────────────────────────────────────────────┘
```

## Стратегии кэширования

| Тип ресурса | Стратегия | Кэш | Описание |
|---|---|---|---|
| HTML (`/`, `/index.html`) | **NetworkFirst** | `vet-app-X.X.X` | Сначала сеть → при ошибке: кэш → offline.html |
| CSS / JS / иконки | **CacheFirst** | `vet-app-X.X.X` | Сначала кэш → при промахе: сеть |
| Google Fonts CSS | **StaleWhileRevalidate** | `vet-fonts-1` | Возвращаем кэш, обновляем в фоне |
| Google Fonts woff2 | **CacheFirst** | `vet-fonts-1` | Шрифты иммутабельны — кэш навсегда |
| API (`/owners`, `/pets`...) | **Bypass** | — | Обрабатывает `app.js` interceptor |

## Что кэшируется

### Service Worker Cache (статика — `vet-app-X.X.X`)
```
/ — app shell
/index.html
/offline.html
/manifest.json
/css/app.css
/js/db.js       — IndexedDB слой
/js/sync.js     — синхронизация
/js/app.js      — fetch interceptor
/js/ui.js       — формы и модалки
/js/pages.js    — рендеринг страниц
/icons/*.png    — иконки
/icons/*.svg
```

### IndexedDB (`vetclinic-pwa`, версия 2)
```
owners      — владельцы животных
pets        — животные (с жизненным циклом)
items       — каталог услуг и препаратов
visits      — приёмы
visit_items — позиции приёмов
vaccinations — вакцинации
staff       — персонал клиники
sync_queue  — очередь ожидающих операций
sync_state  — метаданные (last_sync, device_id)
devices     — зарегистрированные устройства
```

## Как работает офлайн-режим

### Запись данных офлайн
1. Пользователь создаёт владельца/приём → `app.js` перехватывает `fetch()`
2. Запись сохраняется в IndexedDB с `sync_status: "pending"`
3. UI обновляется **мгновенно** (без задержки)
4. `triggerSync()` ставится в очередь с debounce 1.2 сек
5. Если сети нет → `runSync()` пишет в консоль и ждёт события `online`

### Восстановление соединения
1. Браузер генерирует событие `window.online`
2. `app.js` запускает `runSync()`
3. `checkServerReachable()` делает HEAD `/health` → проверяет реальную доступность
4. Если сервер отвечает → `VetSync.syncAll()`
5. Push: все `pending` записи → `POST /sync/push` (батч)
6. Pull: `GET /sync/pull?since=...` → обновить локально

### Background Sync (при закрытом приложении)
1. Пользователь сделал изменения → уходит из приложения
2. Background Sync API регистрирует тег `vetclinic-sync`
3. При появлении сети браузер будит SW
4. SW уведомляет открытые вкладки → они запускают синхронизацию

## Версионирование кэша

Кэш привязан к `APP_VERSION` в `service-worker.js`:

```javascript
var APP_VERSION = "1.4.0";
var CACHE_APP   = "vet-app-" + APP_VERSION;
```

**При обновлении кода:**
1. Измените `APP_VERSION` в `service-worker.js`
2. Задеплойте → браузер обнаружит изменённый SW-файл
3. Новый SW устанавливается в фоне (не прерывая работу)
4. Пользователю показывается тост **«Доступно обновление»**
5. Клик «Обновить» → `SKIP_WAITING` → перезагрузка → новый кэш

## Обнаружение обновлений

```
Пользователь открывает приложение
    ↓
Browser: SW файл изменился? → Загружает новый SW
    ↓
New SW: installs (pre-cache)
    ↓
Старый SW ещё активен → новый ждёт (waiting)
    ↓
app.js: reg.waiting обнаружен → тост «Доступно обновление»
    ↓
Клик «Обновить» → SKIP_WAITING → controllerchange
    ↓
app.js: location.reload() → новый SW активен
```

## Установка на Android (Xiaomi Pad 7)

### Способ 1 — Автоматический баннер
1. Откройте Chrome → `http://ваш-сервер:8080`
2. Загрузите данные (первый визит)
3. Внизу появится баннер «Установить VetClinic»
4. Нажмите **«Установить»** → подтвердите в диалоге Chrome
5. Иконка появится на главном экране

### Способ 2 — Ручная установка
1. Откройте Chrome → меню (⋮) → **«Добавить на главный экран»**
2. Или: меню → **«Установить приложение»**

### Способ 3 — Адресная строка
1. В Chrome 116+ появляется иконка установки в адресной строке
2. Нажмите → «Установить»

### После установки
- Приложение запускается без браузерной строки (полноэкранный режим)
- Работает **полностью офлайн** после первого запуска
- Иконка на главном экране открывает нужную секцию (shortcuts)
- Ориентация `any` — поддерживает и портрет и ландшафт

## Проверка работы офлайн

```bash
# 1. Запустить сервер
go run ./backend/

# 2. Открыть Chrome → localhost:8080
# 3. Убедиться что данные загружены (обзор показывает статистику)

# 4. DevTools → Network → Offline (или отключить сеть в ОС)

# 5. Перезагрузить страницу (F5)
#    → Приложение загружается из SW кэша ✓

# 6. Создать нового владельца / приём
#    → Сохраняется в IndexedDB ✓
#    → Статус синхронизации: "Офлайн" ✓

# 7. Включить сеть обратно
#    → Статус: "Синхронизация..." → "Актуально" ✓
#    → Данные уходят на сервер ✓
```

## Обновление версии

Когда вы изменили код фронтенда:

```javascript
// service-worker.js — строка 1
var APP_VERSION = "1.5.0";  // ← поменяйте версию
```

Старый кэш `vet-app-1.4.0` удалится автоматически при активации нового SW.
