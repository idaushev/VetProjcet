# Телеграм-бот: архитектура

Бот решает две задачи:

1. **Выдача доступов владельцам** — присылает ссылку на портал
   (`https://<сервер>/portal`) и в перспективе одноразовый код входа.
2. **Информирование** — напоминания о следующем приёме, о подходящей
   вакцинации, произвольные сообщения из клиники.

## Что уже заложено в коде (v2.26+)

| Компонент | Где | Что делает |
|---|---|---|
| `owner_telegram` | таблица | привязка владельца к чату: `owner_id → chat_id` |
| `telegram_link_codes` | таблица | одноразовые коды привязки (код → owner_id, срок) |
| `notifications` | таблица | **outbox**: всё, что бот должен отправить |
| `enqueueOwnerNotification()` | `backend/telegram.go` | положить сообщение владельцу в очередь |
| `linkOwnerChat()` | `backend/telegram.go` | привязать чат + дослать накопившееся |
| `startTelegramNotifier()` | `backend/telegram.go` | фоновый отправитель: раз в 30 с доставляет pending-строки через Bot API |
| `TELEGRAM_BOT_TOKEN` | env / `config.go` | токен от @BotFather; пусто — бот спит |

Ключевое свойство: **outbox работает и без бота**. Любой код клиники может
уже сейчас вызывать `enqueueOwnerNotification(ctx, ownerID, kind, message)` —
сообщения лягут в таблицу и уедут, как только появится токен и привязка.

Виды уведомлений (`kind`): `portal_access`, `visit_reminder`,
`vaccination_due`, `custom`.

## Поток привязки владельца (следующий этап)

1. Регистратор в карточке владельца нажимает «Привязать телеграм» → сервер
   создаёт код в `telegram_link_codes` и показывает deep-link
   `https://t.me/<бот>?start=<код>` (QR на экране/чеке).
2. Владелец открывает ссылку → бот получает `/start <код>` через `getUpdates`.
3. Обработчик находит код, вызывает `linkOwnerChat(ownerID, chatID, username)`,
   бот отвечает приветствием и ссылкой на портал (`portal_access`).

## Что осталось реализовать

- [ ] Цикл `getUpdates` (long-poll) + обработка `/start <код>`.
- [ ] Кнопка «Привязать телеграм» в карточке владельца (генерация кода + QR).
- [ ] Планировщик напоминаний: ежедневный проход по `visits.next_visit_date`
      и `vaccinations.next_due_at` → enqueue `visit_reminder` / `vaccination_due`.
- [ ] Вход на портал по коду из телеграма (вместо/вместе с телефоном).

## Запуск

```
set TELEGRAM_BOT_TOKEN=123456:ABC-DEF...   # или переменная окружения в start.bat
```

Без токена сервер пишет в лог «Телеграм-бот выключен» и продолжает работать.
