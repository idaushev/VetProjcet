# Телеграм-бот: архитектура

Бот решает две задачи:

1. **Выдача доступов владельцам** — одноразовый пароль входа на портал
   (`https://<сервер>/portal`); без пароля от бота в кабинет не войти.
2. **Информирование** — напоминания о следующем приёме, о подходящей
   вакцинации, произвольные сообщения из клиники.

## Что уже заложено в коде (v2.26+)

| Компонент | Где | Что делает |
|---|---|---|
| `owner_telegram` | таблица | привязка владельца к чату: `owner_id → chat_id` |
| `telegram_link_codes` | таблица | одноразовые коды привязки (код → owner_id, срок) |
| `notifications` | таблица | **outbox**: всё, что бот должен отправить |
| `enqueueOwnerNotification()` | `backend/telegram.go` | положить сообщение владельцу в очередь |
| `linkOwnerChat()` | `backend/telegram.go` | привязать чат + дослать накопившееся |
| `startTelegramNotifier()` | `backend/telegram.go` | фоновый отправитель: раз в 30 с доставляет pending-строки через Bot API |
| `TELEGRAM_BOT_TOKEN` | env / `config.go` | токен от @BotFather; пусто — бот спит |

Ключевое свойство: **outbox работает и без бота**. Любой код клиники может
уже сейчас вызывать `enqueueOwnerNotification(ctx, ownerID, kind, message)` —
сообщения лягут в таблицу и уедут, как только появится токен и привязка.

Виды уведомлений (`kind`): `portal_access`, `visit_reminder`,
`vaccination_due`, `custom`.

## Вход на портал: пароль от бота (реализовано)

1. Владелец пишет боту (`/start` или что угодно) → бот просит номер телефона
   (кнопка «📱 Отправить мой номер» или текстом).
2. Бот нормализует номер, находит владельца в `owners`, привязывает чат
   (`linkOwnerChat`) и присылает **одноразовый 6-значный пароль**
   (`issuePortalCode`, таблица `portal_codes`): TTL 10 минут, одно
   использование, новый код гасит прежний.
3. Владелец на `/portal` вводит телефон + пароль → `POST /portal/login`
   сверяет код и выдаёт сессию на 90 дней.
4. Новый пароль — написать боту «пароль» (подойдёт любое сообщение).

Приём входящих — long-poll `getUpdates` (`telegramPollUpdates`), offset
хранится в `telegram_state`, ответы диалога уходят сразу (`telegramReply`),
мимо outbox.

## Запуск

```
set TELEGRAM_BOT_TOKEN=123456:ABC-DEF...   # токен от @BotFather (обязателен для входа владельцев)
set TELEGRAM_BOT_NAME=vetclinic_bot        # имя без @ — ссылка на бота на странице входа
set PORTAL_URL=https://192.168.0.10:8443/portal   # бот добавит ссылку к паролю
```

Без токена сервер пишет в лог «Телеграм-бот выключен» и продолжает работать,
но владельцы не смогут получить пароль — вход на портал фактически закрыт.
