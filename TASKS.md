# Бэклог

Приоритизированный список задач по итогам UX-аудита (20.07.2026).
Формат: [приоритет] задача — эффект. Сделанное переносится в «Готово» с версией.

## В работу (по порядку)

### P1 — швы между модулями (низкая цена, высокий эффект)

- [ ] **Следующий приём → запись в расписании.** При сохранении приёма
      с заполненной датой следующего визита предлагать создать запись
      в `appointments` (время/врач из контекста приёма). Закрывает разрыв:
      врач назначил повторный визит, а регистратура его не видит.
- [ ] **Виджет «Записи на сегодня» на дашборде.** Ближайшие 3–5 записей
      (время, питомец, врач) + количество в стат-карточке. Утро смены
      начинается с «Обзора», расписание должно быть видно оттуда.
- [ ] **Запись владельца на портале.** Блок «Ваша запись: завтра 15:00,
      врач X» на первом экране кабинета. Нужен эндпоинт
      GET /portal/appointments (по owner_id сессии). Снижает неявки.

### P2 — контроль денег в отчётах

- [ ] **Сводка скидок.** В «Отчёт за день» и «Выручка за период»:
      строка «Скидок: N на X ₸» + разворачиваемый список
      (врач, сумма, причина). Причины уже собираются (discount_reason) —
      без сводки контроль не работает.
- [ ] **Цена неявок.** В отчёте за период: «Потенциально потеряно
      из-за неявок: ~X ₸» (количество no_show из расписания × средний чек).
      Сделать ДО запуска напоминаний бота — будет базовая линия
      для измерения их эффекта.

### P3 — напоминания телеграм-бота (стратегическое)

- [ ] **Планировщик напоминаний.** Ежедневный проход: записи на завтра
      (appointments, status=scheduled, owner привязан к боту) →
      «завтра вы записаны»; вакцинации недели (next_due_at) →
      «подходит срок прививки». Инфраструктура готова: outbox,
      привязка чатов, расписание (см. docs/TELEGRAM.md).
- [ ] **Объединить источники «не пришли».** Отчёт «Не пришли» разделить:
      «не пришли по записи» (appointments.no_show) и «не вернулись
      на повторный приём» (visits.next_visit_date без последующего визита) —
      это разные проблемы с разными действиями.

### P4 — эксплуатационная гигиена

- [ ] **Пагинация списков владельцев и животных** — перенести паттерн
      «Показать ещё» со списка приёмов (растущая база тормозит планшет).
- [ ] **Рабочие часы расписания в настройки клиники** — сейчас 08:00–20:00
      зашито в код (SCHED_START_H/END_H в pages.js).
- [ ] **Линия «сейчас» в расписании** — красная отметка текущего времени
      в сетке дня.

### P5 — дизайн-система (по итогам дизайн-аудита 20.07.2026)

- [ ] **Типографическая шкала.** В CSS 39 разных font-size (149 объявлений).
      Ввести 7 ступеней токенами (--fs-xs…--fs-3xl) и свести к ним.
- [ ] **Контраст вспомогательного текста.** --text-3 (#93a5b6) даёт ≈2.3:1
      на фоне — ниже WCAG AA (4.5:1). Затемнить до ~#6b7c8d, проверить
      подсказки, даты, подписи полей.
- [ ] **Планшетная навигация.** Сайдбар на планшете скрыт за гамбургером —
      частые переходы стоят два касания. Нижняя панель на 4–5 главных
      разделов (Обзор, Расписание, Приёмы, Поиск, Ещё).
- [ ] **Sticky-кнопка «Сохранить» в форме приёма** — форма ~1.5 экрана
      прокрутки на планшете, кнопка должна быть всегда видна.
- [ ] **Единая иконография.** Смешаны эмодзи (виды животных, поиск в шапке)
      и SVG-штриховые иконки. Эмодзи оставить только как аватары видов,
      остальное — SVG.
- [ ] **Инлайн-стили → классы.** ~196 style="..." в JS-шаблонах
      (pages.js 104, ui.js 47) + 45 в разметке. Постепенно выносить
      в компонентные классы (правило бойскаута: трогаешь блок — выноси).
- [ ] **Пустые состояния с действием.** «Приёмов нет» → короткий текст
      + кнопка следующего шага («Записать», «Создать приём»).
- [ ] **Радиусы-отщепенцы.** Свести сырые 3px/4px/10px/99px
      к токенам --r-sm/--r/--r-full.
